Flow (Order of Operations)

1User sends request to /api/v1/analysis/chat.
2Backend loads/creates AnalysisSession and prepares context.
3Agent loop calls Claude and receives next action/final answer.
4Tool dispatch executes Python/data/knowledge actions.
5Validation layer enforces units and physical plausibility.
6Agent returns answer + react trace + figures + notebook state.
7Client fetches figure PNG / notebook via analysis endpoints.
Design Highlights
• Built on top of MVP: existing /api/v1/chat and AgentService remain unchanged.
• Explicit ReAct loop for auditability and controlled multi-step analysis.
• Pluggable execution backends (subprocess / jupyter / anthropic) selected by config.
• Physical safety checks (units + ranges + hard laws) before final answer.
• Secure local file access using path-resolution guards.
